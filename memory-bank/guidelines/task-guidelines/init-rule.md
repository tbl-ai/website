# Project Initialization Rule

## 1. Understand Project Requirements
- Review the user's request to understand the project goals.
- Identify the core features and requirements.

## 2. Setup Memory Bank
- Create the project-specific folder in `memory-bank/{project-name}`.
- Create `foundation/projectbrief.md` and `foundation/techContext.md`.
- Populate `projectbrief.md` with project goals and requirements.
- Populate `techContext.md` with the proposed tech stack.

## 3. Setup Development Task
- Create a task folder in `memory-bank/{project-name}/develop/{TASK_ID}`.
- Create `activeContext.md` and `progress.md` within the task folder.

## 4. Initial Planning
- Break down the project into smaller tasks.
- Create a `plan.md` file in the task folder, outlining the steps.
- Use the `todo_write` tool to create a trackable task list.
